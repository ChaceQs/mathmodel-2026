# 最优控制 → MILP → MPC：三问题理论统一文档

> **交付对象**：写作手 & 写作手AI助手
> **用途**：论文中需要体现三问题之间"一条逻辑主线贯穿始终"的段落
> **核心目的**：让评委看到三个问题的建模不是各自为战，而是统一的递进框架

---

## 一、统一逻辑主线

```
问题一：特征分析
   │  站点分类 + 关键度评分 + 潮汐规律
   │
   ▼
问题二：等价性论证（理论基石）
   │  最优控制 ≡ 静态 MILP
   │  条件：无骑行扰动 / 仅终端成本 / 路径无关
   │
   ▼
问题三：MPC 滚动时域（条件拓展）
   │  问题二的条件在日际尺度上被打破 → 需要动态适应
   │  但日内 MILP 依然正确 → 作为 MPC 的内核
   │  外层 MPC 处理日际动态 → 内层 MILP 保持静态效率
```

---

## 二、为什么这个统一很重要

大多数参赛论文把三个问题做成"三个独立小程序"。但真正强队的写法是：

> **"本文的三个模型是一个有机整体——问题二的等价性论证为问题三的 MPC 框架提供了理论保证。"**

这会让评委觉得你不只是"做完了题"，而是"理解了题目的结构"。

---

## 三、论文中体现统一的三个位置

### 位置1：论文第一章末尾 — 用一段话穿针引线

> "本文的建模工作遵循一条统一的逻辑主线。问题一通过数据特征分析识别站点功能和潮汐规律，为后续建模提供特征基础。问题二建立了以站点库存为状态、调度量为控制的最优控制模型，严格证明了在调度窗口无骑行需求的条件下该模型等价于混合整数线性规划（MILP），并通过时序恢复验证了动态可执行性。问题三将该等价 MILP 作为内核，在上层构建滚动时域 MPC 框架——以天为周期，用 XGBoost 预测需求作为输入，调用 MILP 求解当日最优调度，执行后反馈实际库存更新状态，滚动推进。三个问题从特征分析到静态最优再到动态自适应，构成了一个完整的'分析—优化—预测与适应'闭环。"

### 位置2：问题二末尾 — 铺垫问题三

> "需要指出，上述等价性依赖于'调度窗口内无骑行需求'这一关键前提。在日际尺度上，当骑行需求逐日变化导致条件失效时，单次静态优化无法应对跨日动态。这一问题将在第三章的滚动时域 MPC 框架中得到解决——每次调用 MILP 求解时等价性依然成立，而日与日之间的动态适应交由外层滚动优化完成。"

### 位置3：问题三开头 — 呼应问题二

> "问题二中证明了最优控制可等价转化为静态 MILP，条件为调度窗口内无外生扰动、成本仅依赖终端状态。在问题三中，当预测模型引入日际需求变化后，本文采用分层策略：将问题二的 MILP 模型作为内层优化引擎（日内等价性依然成立），在外层构建 MPC 滚动时域框架处理日际动态（预测→优化→执行→反馈→更新）。该设计实现了'日内静态等价优化'与'日际动态自适应'的分层协作。"

## 四、模型之间的数据传递

```
问题一 → 问题二：
  站点特征分析结果.csv
  ├── 关键度评分 → 问题二计算调度优先级权重（预留接口）
  └── 站点分类 → 论文定性讨论

问题二 → 问题三：
  MILP 调度模型（作为函数接口）
  ├── solve_milp(I0, I_star, ...) → 最优调度方案 + 总成本
  ├── 问题三的 MPC 框架每天调用此函数
  └── 问题三的方案对比中，方案B调用此函数

问题三 内部：
  XGBoost 预测需求 → MILP 求解调度 → MPC 滚动执行
```

## 六、论文模板段落（可整体复制）

建议在论文的"问题分析"章节末尾，或"模型建立"章节开头，插入以下过渡段：

> 三个问题之间存在紧密的递进逻辑。问题一通过数据挖掘提取站点分类与潮汐规律，为后续建模提供定性与定量依据。问题二以最优控制理论为框架，建立了以站点库存为状态变量、调度量为控制变量的连续时间优化模型，并严格论证了在调度窗口无骑行需求条件下，该模型等价于混合整数线性规划——这一等价性是本文模型体系的理论基石。问题三在此等价性的基础上向外拓展：当日际需求变化使条件（1）在更长的时间尺度上不再成立时，本文将等价的 MILP 作为内层优化引擎，封装于外层的滚动时域 MPC 框架之中。该分层设计既保留了静态优化的求解效率与全局最优性，又通过每日的"预测→优化→执行→反馈"循环实现了对实际运营动态的自适应响应。三个模型共同构成了一个从"理解系统"到"静态优化"再到"动态适应"的完整分析链。

---

## 七、一句话总结

**问题二的等价性论证是整个模型体系的"地基"——上面既可以直接盖静态优化（问题二），也可以向上叠加 MPC 框架（问题三）。在论文中把这条线讲清楚，三题就不再是三个独立的答案，而是一篇有灵魂的论文。**
