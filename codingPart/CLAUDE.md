## Project Overview

This is a mathematical modeling project for the 2026 Henan University Mathematical Modeling Competition, focused on bike-sharing dispatch optimization. The project analyzes bike station data, identifies patterns (tidal phenomena, weekday/weekend differences, weather impacts), and performs station clustering to optimize dispatch strategies.

The project consists of two main scripts that should be run in sequence:


## Data Files

The project uses CSV data files with Chinese headers:
- `站点基础信息.csv` - Station basic information (ID, name, type, capacity)
- `每日初始库存.csv` - Daily initial inventory per station
- `天气数据.csv` - Weather data (date, weather type, holiday flag)
- `调度记录表.csv` - Dispatch records (source/destination stations, time windows)
- `站点期望库存.csv` - Expected inventory levels per station
- `站点距离.csv` - Distance matrix between stations
- `日汇总骑行数据.csv` - Daily riding summary (inflow/outflow per station)

## Code Architecture

### problem1_analysis.py
Main analysis pipeline with five sections:
1. **Data Loading & Quality Assessment** - Reads all CSV files, checks for missing values and data types
2. **Logical Consistency Validation** - Verifies inventory changes match riding + dispatch records, checks capacity constraints
3. **Temporal-Spatial Feature Analysis** - Analyzes station flow patterns, tidal phenomena (morning/evening peaks), weekday vs weekend differences, weather impacts
4. **Station Clustering** - Uses K-means clustering (K=3) on standardized features to classify stations into three types:
   - Supply-type stations (residential areas, net outflow)
   - Demand-type stations (business districts, net inflow)
   - Balanced stations (transit hubs)
5. **Key Station Identification** - Calculates importance scores based on flow volume, tidal intensity, and dispatch frequency

**Output**: `站点特征分析结果.csv` containing all computed features and cluster labels

## Key Analysis Concepts

**Tidal Phenomenon (潮汐现象)**: The pattern where residential stations have net outflow during morning rush hours and net inflow during evening rush hours, while business districts show the opposite pattern.

**Tidal Index (潮汐指数)**: Calculated as `abs(morning_net_outflow) + abs(evening_net_inflow)`, measuring the strength of directional flow patterns.

**Key Station Score (关键度评分)**: Weighted combination of average outflow (30%), average inflow (30%), tidal index (20%), and absolute net dispatch frequency (20%).

## Data Validation

The analysis includes comprehensive validation:
- Inventory consistency: `next_day_inventory = current_inventory + net_riding_flow + net_dispatch`
- Capacity constraints: All inventory values must be ≤ station capacity
- Time window compliance: All dispatch operations must occur between 00:00-05:00
