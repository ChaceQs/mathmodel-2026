# 问题三：共享单车需求预测模型设计

## 一、问题分析

### 1.1 核心任务
- 使用前10天数据（6月1日-6月10日）训练预测模型
- 预测未来2天（6月11日-6月14日）各站点的需求变化
- 将预测结果集成到问题二的调度优化模型中
- 对比"预测需求"与"历史平均需求"的调度效果

### 1.2 实际数据规模
- 站点数量：N = 10
- 时间跨度：14天（2023.6.1-6.14）
- 训练集：前10天
- 测试集：后4天
- 历史调度记录：61条
- 骑行数据：140条

### 1.3 需求预测的关键因素
1. **时间因素**：工作日/周末、早晚高峰、节假日
2. **空间因素**：站点类型（居民区/商务区/交通枢纽）、站点间距离
3. **环境因素**：天气、温度、降雨
4. **历史模式**：潮汐现象、周期性规律
5. **站点间转移**：骑行OD矩阵、流量守恒

## 二、建模方案对比

### 2.1 微分方程方法 vs AI方法

| 维度 | 微分方程方法 | AI方法 |
|------|------------|--------|
| **理论基础** | 物理规律、系统动力学 | 数据驱动、统计学习 |
| **数据需求** | 中等（需要参数标定） | 高（需要大量训练数据） |
| **可解释性** | 强（物理意义明确） | 弱（黑盒模型） |
| **预测精度** | 中等（依赖模型假设） | 高（捕捉复杂模式） |
| **计算复杂度** | 低（求解ODE/PDE） | 高（训练阶段）低（推理阶段） |
| **外推能力** | 强（基于机理） | 弱（依赖训练分布） |
| **适用场景** | 机理清晰、数据少 | 数据充足、模式复杂 |

### 2.2 推荐策略

**本问题特点**：
- 数据量：10天×10站点=100个观测（中等规模）
- 机理性：单车流动遵循守恒定律（机理清晰）
- 复杂性：受多因素影响（天气、时间、空间）
- 目标：预测未来2-4天（短期预测）

**推荐方案**：**混合建模**
1. **主模型**：AI方法（LSTM/XGBoost）- 捕捉复杂模式
2. **辅助模型**：微分方程 - 提供物理约束
3. **集成策略**：加权融合或约束优化

## 三、微分方程建模方案

### 3.1 基于流体力学的连续模型

**核心思想**：将单车流动类比为流体流动，站点库存类比为流体密度。

#### 数学模型

**守恒方程**：
```
∂ρᵢ(t)/∂t = Σⱼ φⱼᵢ(t) - Σⱼ φᵢⱼ(t) + Rᵢ(t)
```

其中：
- `ρᵢ(t)`：站点i在时刻t的单车密度（库存量）
- `φᵢⱼ(t)`：从站点i到站点j的流量（骑行需求）
- `Rᵢ(t)`：站点i的调度补充量

**流量模型**（类比Fick扩散定律）：
```
φᵢⱼ(t) = Dᵢⱼ · (ρᵢ(t) - ρⱼ(t)) + Fᵢⱼ(t)
```

其中：
- `Dᵢⱼ`：扩散系数（站点间转移倾向）
- `Fᵢⱼ(t)`：外部驱动力（潮汐效应）

**潮汐驱动力**：
```
Fᵢⱼ(t) = Aᵢⱼ · sin(2π·t/T + φᵢⱼ)
```

其中：
- `Aᵢⱼ`：振幅（潮汐强度）
- `T`：周期（24小时）
- `φᵢⱼ`：相位（早晚高峰时间）

#### 参数估计

1. **扩散系数** `Dᵢⱼ`：通过历史骑行OD矩阵估计
   ```
   Dᵢⱼ = (骑行次数ᵢⱼ) / (平均库存差)
   ```

2. **潮汐参数** `Aᵢⱼ, φᵢⱼ`：通过傅里叶分析提取周期性成分

3. **初始条件**：`ρᵢ(0)` = 当日初始库存

#### 求解方法

- **数值方法**：四阶Runge-Kutta法
- **时间步长**：Δt = 1小时
- **预测时域**：24-96小时

#### 优缺点

**优点**：
- 物理意义明确，符合守恒定律
- 参数少，易于标定
- 可解释性强

**缺点**：
- 假设连续流动，实际是离散事件
- 难以捕捉非线性效应
- 对突发事件（天气变化）响应不足

### 3.2 基于传染病模型的离散模型

**核心思想**：将单车需求类比为"传染病传播"，站点间相互影响。

#### SIR改进模型

将站点分为三类：
- **S（Susceptible）**：供给型站点（单车过剩）
- **I（Infected）**：需求型站点（单车短缺）
- **R（Recovered）**：平衡型站点（供需平衡）

**状态转移方程**：
```
dSᵢ/dt = -β·Sᵢ·Σⱼ Iⱼ/dᵢⱼ + γ·Rᵢ
dIᵢ/dt = β·Sᵢ·Σⱼ Iⱼ/dᵢⱼ - α·Iᵢ
dRᵢ/dt = α·Iᵢ - γ·Rᵢ
```

其中：
- `β`：传播率（需求扩散速度）
- `α`：恢复率（调度响应速度）
- `γ`：失衡率（平衡被打破速度）
- `dᵢⱼ`：站点间距离

#### 参数标定

1. **传播率** `β`：通过历史需求扩散速度估计
   ```
   β = Δ需求量 / (供给站点数 × 需求站点数 × Δt)
   ```

2. **恢复率** `α`：通过历史调度响应时间估计
   ```
   α = 1 / 平均调度响应时间
   ```

3. **失衡率** `γ`：通过历史平衡状态持续时间估计

#### 优缺点

**优点**：
- 捕捉站点间相互影响
- 模拟需求传播过程
- 适合网络结构分析

**缺点**：
- 类比不够精确（单车≠传染病）
- 参数物理意义不够直观
- 难以处理多时间尺度问题

### 3.3 基于系统动力学的状态空间模型

**核心思想**：将单车系统建模为动态系统，用状态空间方程描述。

#### 状态空间表示

**状态向量**：
```
X(t) = [ρ₁(t), ρ₂(t), ..., ρₙ(t), v₁(t), v₂(t), ..., vₙ(t)]ᵀ
```

其中：
- `ρᵢ(t)`：站点i的库存量
- `vᵢ(t)`：站点i的库存变化率

**状态方程**：
```
dX(t)/dt = A·X(t) + B·U(t) + C·W(t)
```

其中：
- `A`：系统矩阵（站点间耦合关系）
- `B`：控制矩阵（调度影响）
- `C`：扰动矩阵（外部因素）
- `U(t)`：控制输入（调度决策）
- `W(t)`：外部扰动（天气、时间）

**观测方程**：
```
Y(t) = H·X(t) + V(t)
```

其中：
- `Y(t)`：观测值（实际库存）
- `H`：观测矩阵
- `V(t)`：观测噪声

#### 系统矩阵设计

**A矩阵**（站点间耦合）：
```
Aᵢⱼ = {
  -Σₖ λᵢₖ,  if i = j  (流出总和)
  λⱼᵢ,      if i ≠ j  (从j到i的流入率)
}
```

其中 `λᵢⱼ` 是从站点i到站点j的转移率，通过历史数据估计：
```
λᵢⱼ = (骑行次数ᵢⱼ) / (站点i平均库存 × 观测时长)
```

**B矩阵**（调度影响）：
```
Bᵢⱼ = {
  1,   if 调度车辆j服务站点i
  0,   otherwise
}
```

**C矩阵**（外部因素）：
```
Cᵢₖ = 站点i对外部因素k的敏感度
```

通过回归分析估计：
```
Cᵢₖ = ∂ρᵢ/∂wₖ
```

#### 卡尔曼滤波预测

**预测步骤**：
1. **状态预测**：`X̂(t+1|t) = A·X̂(t|t) + B·U(t)`
2. **协方差预测**：`P(t+1|t) = A·P(t|t)·Aᵀ + Q`
3. **卡尔曼增益**：`K(t+1) = P(t+1|t)·Hᵀ·[H·P(t+1|t)·Hᵀ + R]⁻¹`
4. **状态更新**：`X̂(t+1|t+1) = X̂(t+1|t) + K(t+1)·[Y(t+1) - H·X̂(t+1|t)]`
5. **协方差更新**：`P(t+1|t+1) = [I - K(t+1)·H]·P(t+1|t)`

其中：
- `Q`：过程噪声协方差
- `R`：观测噪声协方差

#### 参数估计

1. **转移率矩阵A**：通过历史骑行OD数据估计
2. **噪声协方差Q, R**：通过最大似然估计或EM算法
3. **初始状态** `X̂(0|0)`：使用历史平均值
4. **初始协方差** `P(0|0)`：使用历史方差

#### 优缺点

**优点**：
- 理论框架完整（控制论基础）
- 可处理不确定性（卡尔曼滤波）
- 可实时更新（在线学习）
- 可与优化模型无缝集成

**缺点**：
- 假设线性系统（实际可能非线性）
- 假设高斯噪声（实际可能非高斯）
- 参数矩阵维度高（N×N）
- 需要较多历史数据标定

#### 扩展：非线性版本

**扩展卡尔曼滤波（EKF）**：
```
dX(t)/dt = f(X(t), U(t), W(t))
Y(t) = h(X(t)) + V(t)
```

通过雅可比矩阵线性化：
```
A(t) = ∂f/∂X |_{X=X̂(t)}
H(t) = ∂h/∂X |_{X=X̂(t)}
```

**无迹卡尔曼滤波（UKF）**：
- 使用sigma点采样
- 避免雅可比矩阵计算
- 对强非线性系统更鲁棒

## 四、AI预测建模方案

### 4.1 时间序列预测模型

#### 4.1.1 ARIMA模型

**模型形式**：
```
ARIMA(p, d, q): (1-Σφᵢ·Lⁱ)·(1-L)ᵈ·yₜ = (1+Σθⱼ·Lʲ)·εₜ
```

其中：
- `p`：自回归阶数
- `d`：差分阶数
- `q`：移动平均阶数
- `L`：滞后算子

**针对本问题**：
- 对每个站点建立独立的ARIMA模型
- 使用AIC/BIC准则选择最优(p,d,q)
- 考虑季节性：SARIMA(p,d,q)(P,D,Q)ₛ，s=7（周周期）

**参数选择**（基于10天数据）：
```python
# 站点i的ARIMA模型
for i in range(10):
    # 平稳性检验（ADF检验）
    if not stationary(data[i]):
        d = 1  # 一阶差分
    
    # ACF/PACF图确定p,q
    p = 根据PACF截尾确定
    q = 根据ACF截尾确定
    
    # 拟合模型
    model = ARIMA(data[i], order=(p,d,q))
    model.fit()
```

**优缺点**：
- ✅ 理论成熟，易于实现
- ✅ 适合单变量时间序列
- ❌ 无法利用站点间关系
- ❌ 无法融入外部变量（天气）
- ❌ 数据量少时不稳定

#### 4.1.2 VAR模型（向量自回归）

**模型形式**：
```
Yₜ = c + A₁·Yₜ₋₁ + A₂·Yₜ₋₂ + ... + Aₚ·Yₜ₋ₚ + εₜ
```

其中：
- `Yₜ = [y₁ₜ, y₂ₜ, ..., y₁₀ₜ]ᵀ`：10个站点的库存向量
- `Aᵢ`：10×10系数矩阵
- `εₜ`：误差向量

**优势**：
- 捕捉站点间动态关系
- 考虑空间相关性
- 可进行格兰杰因果检验

**参数估计**：
```python
# 选择滞后阶数p
p_optimal = select_order(data, maxlags=5, ic='aic')

# 拟合VAR模型
model = VAR(data)
results = model.fit(p_optimal)

# 预测未来4天
forecast = results.forecast(data[-p:], steps=4)
```

**挑战**：
- 参数数量：10×10×p（p=3时，300个参数）
- 样本量：仅10天数据
- 解决方案：正则化VAR、稀疏VAR

#### 4.1.3 Prophet模型

**模型组成**：
```
y(t) = g(t) + s(t) + h(t) + εₜ
```

其中：
- `g(t)`：趋势项（分段线性或逻辑增长）
- `s(t)`：季节性项（傅里叶级数）
- `h(t)`：节假日效应
- `εₜ`：误差项

**季节性建模**：
```
s(t) = Σₙ [aₙ·cos(2πnt/P) + bₙ·sin(2πnt/P)]
```

其中P=7（周周期）

**优势**：
- 自动处理缺失值
- 自动检测趋势变化点
- 易于加入外部变量（天气）
- 鲁棒性强

**实现**：
```python
from fbprophet import Prophet

# 对每个站点
for station_id in range(10):
    df = prepare_data(station_id)  # 日期+库存
    
    # 添加外部变量
    df['weather'] = weather_data
    df['is_weekend'] = is_weekend
    
    # 训练模型
    model = Prophet(
        yearly_seasonality=False,
        weekly_seasonality=True,
        daily_seasonality=False
    )
    model.add_regressor('weather')
    model.add_regressor('is_weekend')
    model.fit(df)
    
    # 预测
    future = model.make_future_dataframe(periods=4)
    forecast = model.predict(future)
```

### 4.3 深度学习预测模型

#### 4.3.1 LSTM模型

**网络结构**：
```
输入层 → LSTM层1 → Dropout → LSTM层2 → Dropout → 全连接层 → 输出层
```

**数学表达**：
```
# LSTM单元
fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf)  # 遗忘门
iₜ = σ(Wi·[hₜ₋₁, xₜ] + bi)  # 输入门
C̃ₜ = tanh(WC·[hₜ₋₁, xₜ] + bC)  # 候选记忆
Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ  # 更新记忆
oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo)  # 输出门
hₜ = oₜ⊙tanh(Cₜ)  # 隐藏状态
```

**模型实现**：

#### 4.3.2 GRU模型（简化版LSTM）

**GRU单元**：
```
zₜ = σ(Wz·[hₜ₋₁, xₜ])  # 更新门
rₜ = σ(Wr·[hₜ₋₁, xₜ])  # 重置门
h̃ₜ = tanh(W·[rₜ⊙hₜ₋₁, xₜ])  # 候选隐藏状态
hₜ = (1-zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ  # 更新隐藏状态
```

**优势**：
- 参数更少（比LSTM少25%）
- 训练更快
- 小数据集上表现更好

---

#### 4.3.3 Transformer模型

**自注意力机制**：
```
Attention(Q,K,V) = softmax(QKᵀ/√dₖ)·V
```

**多头注意力**：
```
MultiHead(Q,K,V) = Concat(head₁,...,headₕ)·Wᴼ
where headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)
```

**优势**：
- 并行计算（比LSTM快）
- 捕捉长距离依赖
- 可解释性（注意力权重）

**挑战**：
- 需要更多数据
- 计算资源需求高
- 本问题数据量可能不足

---

#### 4.3.4 图神经网络（GNN）

**核心思想**：将站点建模为图节点，站点间关系建模为边。

**图结构**：
```
节点：10个站点
边：站点间距离 < 阈值 或 历史骑行次数 > 阈值
节点特征：库存、容量、类型
边特征：距离、历史骑行次数
```

**GCN传播规则**：
```
H^(l+1) = σ(D̃^(-1/2)·Ã·D̃^(-1/2)·H^(l)·W^(l))
```

其中：
- `Ã = A + I`：邻接矩阵+自环
- `D̃`：度矩阵
- `H^(l)`：第l层节点特征
- `W^(l)`：可学习权重

**优势**：
- 显式建模空间关系
- 捕捉站点间相互影响
- 适合网络结构数据

**实现框架**：PyTorch Geometric, DGL

## 五、预测-调度一体化框架

### 5.1 两阶段框架（推荐）

**核心思想**：预测和调度分离，先预测需求，再基于预测结果优化调度。

#### 框架流程

```
阶段1：需求预测
  输入：前10天历史数据 + 外部因素（天气）
  模型：XGBoost / LSTM / 状态空间模型
  输出：未来4天各站点期望库存 Î*ᵢ(t)
  
阶段2：调度优化
  输入：预测的期望库存 Î*ᵢ(t) + 当前库存 I⁰ᵢ
  模型：MILP优化模型（问题二）
  输出：最优调度方案 x*ᵢⱼₖ
```

#### 数学表达

**阶段1：预测模型**
```
Î*ᵢ(t+h) = f_pred(Xᵢ(t), θ)
```

其中：
- `h`：预测时域（1-4天）
- `Xᵢ(t)`：站点i的特征向量
- `θ`：模型参数

**阶段2：优化模型**
```
min Z = Σᵢ wᵢ|Iᵢ - Î*ᵢ| + Σₖᵢⱼ cᵢⱼₖ·xᵢⱼₖ

s.t.  Iᵢ = I⁰ᵢ + Σⱼₖ xⱼᵢₖ - Σⱼₖ xᵢⱼₖ
      0 ≤ Iᵢ ≤ Cᵢ
      Σᵢⱼ xᵢⱼₖ ≤ Qₖ
      Σᵢⱼ tᵢⱼₖ·yᵢⱼₖ ≤ T
```

#### 不确定性处理

**方法1：点预测**
- 使用预测均值作为期望库存
- 简单直接，但忽略不确定性

**方法2：区间预测**
```
Î*ᵢ(t) ∈ [Î*ᵢ,lower, Î*ᵢ,upper]
```

- 使用预测区间的中点
- 或使用鲁棒优化处理区间

**方法3：情景分析**
```
生成K个需求情景：{Î*ᵢ,₁, Î*ᵢ,₂, ..., Î*ᵢ,ₖ}
对每个情景求解优化模型
选择平均性能最好的方案
```

#### 滚动时域策略

**MPC（模型预测控制）框架**：
```
for t = 1 to T:
    # 1. 预测未来H天需求
    forecast = predict(data[t-10:t], horizon=H)
    
    # 2. 求解优化问题
    schedule = optimize(forecast, current_inventory)
    
    # 3. 执行第一天的调度方案
    execute(schedule[0])
    
    # 4. 观测实际结果
    observe(actual_demand[t])
    
    # 5. 更新模型（在线学习）
    update_model(actual_demand[t])
```

**优势**：
- 反馈校正（实时调整）
- 滚动优化（持续改进）
- 鲁棒性强（应对不确定性）

#### 实施步骤

1. **离线训练**（使用前10天数据）
   ```python
   # 训练预测模型
   ```

### 5.2 端到端框架

**核心思想**：将预测和调度整合为一个端到端模型，直接从历史数据输出调度方案。

#### 深度强化学习框架

**MDP建模**：
```
状态 S: {当前库存, 历史需求, 天气, 时间}
动作 A: {调度方案 xᵢⱼₖ}
奖励 R: -（库存偏差 + 调度成本）
```

**Actor-Critic架构**：
```
Actor网络：π(a|s; θ_π) → 输出调度策略
Critic网络：V(s; θ_v) → 评估状态价值
```

**训练流程**：
```python
for episode in range(num_episodes):
    state = env.reset()  # 初始化状态
    
    for t in range(horizon):
        # Actor选择动作
        action = actor.select_action(state)
        
        # 环境反馈
        next_state, reward, done = env.step(action)
        
        # Critic评估
        value = critic.evaluate(state)
        next_value = critic.evaluate(next_state)
        
        # 计算优势函数
        advantage = reward + gamma * next_value - value
        
        # 更新Actor
        actor_loss = -log_prob(action) * advantage
        actor.update(actor_loss)
        
        # 更新Critic
        critic_loss = (reward + gamma * next_value - value)^2
        critic.update(critic_loss)
        
        state = next_state
```

**优势**：
- 端到端学习（无需手工设计特征）
- 自适应能力强（持续学习）
- 可超越历史方案

**挑战**：
- 需要大量数据（10天不足）
- 训练不稳定
- 可解释性差
- 解决方案：模拟环境生成数据、迁移学习

### 5.3 混合框架（推荐）

**核心思想**：结合两阶段框架的可靠性和端到端框架的灵活性。

#### 架构设计

```
层1：预测层（AI模型）
  ├─ XGBoost：捕捉非线性关系
  ├─ LSTM：捕捉时间依赖
  └─ 状态空间模型：提供物理约束

层2：融合层
  └─ 加权平均 / Stacking / 贝叶斯模型平均

层3：优化层（MILP）
  ├─ 确定性优化（点预测）
  ├─ 鲁棒优化（区间预测）
  └─ 随机优化（情景分析）

层4：反馈层
  └─ 在线学习 / 模型更新
```

#### 集成策略

**预测集成**：
```
Î*ᵢ = w₁·Î*ᵢ,XGB + w₂·Î*ᵢ,LSTM + w₃·Î*ᵢ,SSM

权重选择：
- 等权重：w₁ = w₂ = w₃ = 1/3
- 性能加权：wⱼ ∝ 1/MSEⱼ
- 动态权重：根据最近性能调整
```

**约束集成**：
```
# AI预测提供目标
target = AI_predictor(data)

# 物理模型提供约束
constraints = Physics_model(data)

# MILP优化
solution = MILP_solver(
    objective=minimize_deviation(target),
    constraints=constraints
)
```

```
MAE = (1/N)·Σᵢ|ŷᵢ - yᵢ|
RMSE = √[(1/N)·Σᵢ(ŷᵢ - yᵢ)²]
MAPE = (100%/N)·Σᵢ|（ŷᵢ - yᵢ)/yᵢ|
R² = 1 - Σᵢ(yᵢ - ŷᵢ)²/Σᵢ(yᵢ - ȳ)²
```

**2. 区间预测指标**：
```
覆盖率 = (区间包含真实值的样本数) / 总样本数
区间宽度 = 平均(upper - lower)
```

**3. 方向预测指标**：
```
方向准确率 = (预测变化方向正确的样本数) / 总样本数
```
