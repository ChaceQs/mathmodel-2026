# 2026-05-03 问题二数学模型设计（第一版）

## 本次任务：
为问题二设计完整的数学模型，包括变量定义、目标函数、约束条件、数学性质分析

## 关键结论/产出：

### 1. 模型类型
**混合整数线性规划（MILP）**
- 决策变量：$x_{ijk}$（调度量，整数）、$y_{ijk}$（是否调度，二进制）
- 目标函数：多目标加权（库存偏差 + 调度成本 + 调度次数）
- 约束条件：10类线性约束

### 2. 核心数学表达

#### 目标函数
$$\min Z = \alpha \cdot \sum_{i} w_i(\xi_i^+ + \xi_i^-) + \beta \cdot \sum_{k,i,j} y_{ijk}(c_{fuel} \cdot d_{ij} + c_{time} \cdot t_{ij}) + \gamma \cdot \sum_{k,i,j} y_{ijk}$$

#### 关键约束
- **库存守恒**：$I_i = I_i^0 + \sum_{k,j} x_{jik} - \sum_{k,j} x_{ijk}$
- **容量约束**：$0 \leq I_i \leq C_i$
- **载重约束**：$x_{ijk} \leq Q_k \cdot y_{ijk}$
- **时间窗口**：$\sum_{i,j} y_{ijk} \cdot t_{ij} \leq 300$ 分钟

### 3. 数学性质分析

| 性质 | 结论 | 说明 |
|------|------|------|
| **模型类型** | MILP | 线性目标+线性约束+整数变量 |
| **凸性** | 连续松弛是凸问题 | 整数约束破坏凸性 |
| **复杂度** | NP-Hard | 但中小规模可精确求解 |
| **可分解性** | 部分可分解 | 不同卡车任务相对独立 |
| **特殊结构** | 网络流+VRP变体 | 可利用专用算法加速 |
| **问题规模** | $2N^2M + 3N$ 个变量 | N=10, M=3 → 630变量 |

### 4. 求解策略建议
- **精确算法**：Gurobi/CPLEX（推荐用于N≤50）
- **启发式算法**：贪心+局部搜索（大规模问题）
- **混合策略**：启发式生成初始解 + 求解器优化

### 5. 模型创新点
1. 多目标平衡（服务质量+经济性+操作性）
2. 加权偏差（基于问题一的关键度评分）
3. 软约束处理（偏差变量线性化）
4. 时间窗口精确建模
5. 可扩展框架（支持多日、多车型）

### 6. 与问题一的衔接

- 问题一的站点聚类 → 识别供给型/需求型站点

# 2026-05-03 问题二多模型对比分析（完成）

## 本次任务：
设计多种备选模型方案并进行全面对比分析，推荐最佳建模方案

## 关键结论/产出：

### 1. 四种备选模型方案

#### 方案1：混合整数线性规划（MILP）
- **核心**：精确优化的标准方法
- **优势**：理论最优性保证、约束处理严格、可解释性强
- **劣势**：计算复杂度高、大规模问题困难
- **适用**：中小规模、对最优性要求高

#### 方案2：两阶段启发式模型
- **核心**：分解为匹配问题+路径优化
- **阶段1**：贪心匹配生成调度对
- **阶段2**：节约算法优化路径
- **优势**：计算速度快、可处理大规模、易实现
- **劣势**：无最优性保证、可能局部最优
- **适用**：大规模问题、快速求解

#### 方案3：多目标优化模型（帕累托最优）
- **核心**：不预设权重，求解帕累托前沿
- **方法**：ε-约束法、加权法、NSGA-II
- **优势**：避免权重主观性、决策灵活、创新性强
- **劣势**：计算量大、需多次求解
- **适用**：权重难确定、需多方案对比

#### 方案4：网络流优化模型
- **核心**：最小费用最大流问题
- **优势**：求解速度快、理论基础扎实、自然整数解
- **劣势**：难以建模时间窗口、目标单一
- **适用**：网络流结构明显、单目标优化

### 1. 数据预处理策略

**数据质量评估**：

**OLS估计**：
- 正规方程：$\hat{\beta} = (X^TX)^{-1}X^Ty$

- 似然函数最大化
- 泊松回归、负二项回归

### 3. 强化学习方案

**MDP建模**：
```python
状态空间 S: {库存, 期望库存, 时间, 卡车位置, 卡车容量}
动作空间 A: {选择卡车k, 从站点i, 到站点j, 调度量x}
奖励函数 R: -(α×库存偏差 + β×调度成本 + γ×时间惩罚)
状态转移 P: 确定性转移 + 随机性（需求波动）
```

**算法选择**：
1. **Q-Learning**：离散动作空间
2. **DQN**：深度Q网络 + Experience Replay + Target Network
3. **Actor-Critic**：策略网络 + 价值网络
4. **PPO**（推荐）：稳定性好、样本效率高

**实施策略**：
